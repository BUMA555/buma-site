Quick Paths

按问题直达

如果你现在已经卡住了，就别从头翻了，直接进对应问题页。先把入口找对，排查速度会快很多。

飞书接入从哪开始

适合第一次接 OpenClaw + 飞书，先看应用、凭证、Gateway、测试顺序。

权限都开了，为什么还是不工作

适合排查 im.message.receive_v1、发送权限、发布状态和 Bot 能力边界。

飞书群里为什么不回复

适合排查为什么要 @ 才回、groupPolicy、requireMention 和触发范围。

长连接、配对码、群聊触发不通

适合排查 WebSocket、pairing 审批、群聊白名单和事件订阅。

Chrome 扩展显示 ! 或 attach tab 没反应

适合排查 browser relay、chrome profile、ON 徽章和 relay 地址。

Dashboard 打不开、1008、unauthorized

适合先拆清控制台地址、gateway status、鉴权和配对问题。

Dashboard / Gateway

控制台、服务状态和日志排障入口

如果你卡的不是“飞书接不进来”，而是 Dashboard 打不开、Gateway 启不来、日志看不懂、页面反复报 unauthorized，优先从这组入口开始。它们属于同一类问题，但排查顺序并不一样。

Dashboard 打不开、1008、PAIRING_REQUIRED

先拆清控制台地址、openclaw dashboard、openclaw gateway status、鉴权和 pairing 的先后顺序，避免把控制台打不开误判成飞书问题。

OpenClaw logs 怎么看

适合先学会看 openclaw logs --follow、状态面板和常见日志信号，把“到底是哪一层坏了”快速拆出来。

Gateway 起不来怎么办

适合排查 gateway.mode=local、严格配置校验、非 loopback 鉴权限制、EADDRINUSE 和端口占用。

unauthorized / AUTH_TOKEN_MISMATCH 怎么办

适合拆清 AUTH_TOKEN_MISMATCH、AUTH_DEVICE_TOKEN_MISMATCH、设备 token 漂移和 Dashboard 鉴权层的区别。

Top Questions

最常见的 4 类 FAQ

最近高频问题基本都集中在这 4 类。先看自己属于哪一类，再继续深挖。

1. 接入类：到底应该先做什么

  • 先确认 Node、Gateway 和 Dashboard 正常。
  • 再创建飞书应用、填 App ID / Secret。
  • 然后核对事件订阅、Bot 权限和 pairing。
  • 最后再去看群聊规则、定时任务和协作链路。

先看：OpenClaw 飞书机器人接入清单

2. 权限类：看起来都开了，为什么还是发不出去

  • 常见不是“没开权限”，而是开错位置或版本没发布。
  • 飞书事件订阅、Bot 能力和消息发送权限要一起看。
  • 群聊里能不能回，还会受 groupPolicy 和 allowFrom 影响。
  • 排查时别只盯一个报错，要把权限、发布状态和触发链路一起看。

先看：OpenClaw 飞书机器人权限清单

3. 群聊类：为什么私聊能回，群里却经常没反应

  • 默认安全策略通常比想象中更严格。
  • 很多场景下群聊就是要求明确 @，避免误触发。
  • pairing 没通过、群未放行、触发人不在白名单，也会像“没回复”。
  • 这类问题要同时看 requireMention、groupPolicy 和 allowFrom。

先看：OpenClaw 飞书群里为什么不回复

4. 控制台与自动化类：页面打不开、定时任务不触发

  • Dashboard 问题先拆成服务状态、鉴权、配对三层。
  • Gateway 问题再继续拆成配置模式、端口占用、认证前提三层。
  • 日志问题先看 openclaw logs --follow，再回到对应模块处理。
  • 别把所有报错混成一个问题看，分层后更容易处理。

先看：OpenClaw Dashboard 打不开怎么办

继续往下看什么

如果你不是只想排 1 个点，而是想把 OpenClaw 的接入、协同和使用链路整体看清，可以沿着这条顺序看。

第 1 步：飞书机器人接入清单

先把接入顺序、基础配置和第一轮测试跑通。

第 2 步：权限与事件订阅

把“有配置但不生效”的高频误区一次理顺。

第 3 步：群聊触发规则

把为什么经常要 @ 才回、哪些人能触发讲清楚。

第 4 步：先学会看日志

把日志信号和状态检查顺序先理顺，再去处理具体模块。

第 5 步：服务起不来先查 Gateway

把模式、鉴权、端口与配置问题拆开，不要和渠道问题混看。

第 6 步：unauthorized 再看 token

把 Dashboard 鉴权、设备 token 漂移和 pairing 边界讲清楚。
