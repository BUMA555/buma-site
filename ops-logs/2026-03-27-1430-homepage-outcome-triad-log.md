# OPS LOG: 一人公司主站 UI 持续优化（2026-03-27 14:30）

## 一、本轮目标

**本轮只做 1 个小任务**：在首页补一个“结果三点聚焦”的模块，明确用户会获得的 3 个关键结果，降低“我到底能得到什么”的理解成本。

## 二、快速学习了什么

**参考页面 1：Linear 首页**
- 看到的点：价值表达非常收敛，常用“3 条核心价值”形成快速扫读结构。
- 可复用点：在长页面中插入“三点结果聚焦”模块，强化主线记忆与转化锚点。

## 三、本轮落地改动

**改动文件**：
- `site-draft/index.html`

**实际改了什么**：
1. 在首页 `What We Do` 与 `Fit Check` 之间新增 `Outcome` 模块。
2. 用 3 张卡片表达“线索入口清晰 / 执行流程可复制 / 优先级不混乱”。
3. 文案强调“结果”，避免仅罗列功能。

## 四、为什么这轮这样改

- 商业站需要让访客快速理解“我能得到什么结果”，否则会把功能当成信息噪音。
- 三点聚焦结构有助于提高扫读效率和记忆率，提升点击 CTA 的确定感。

## 五、验收证据

**已看参考**：
- `https://linear.app/`

**本地可见证据**：
- `index.html` 中新增注释块：`<!-- OUTCOME -->`
- 新增标题：`先把最关键的 3 个结果讲清楚`
- 新增三张结果卡片：`线索入口变清晰 / 执行流程可复制 / 优先级不再混乱`

## 六、风险

**本方案风险**：结果表达仍为主观承诺，可能被理解为“已保证实现”；若后续反馈觉得夸大，则把文案改为“你将更容易做到/更接近达成”的语气，并补一条案例支持。

## 七、下一步

下一轮优先考虑其中一个小任务：
1. 给首页 CTA 区补“提交后你会收到什么”的明确说明。
2. 文章中心加“按问题分流”的入口，减少纯列表感。
3. 联系页补“准备材料清单”与“预期节奏”。

## 八、阻塞情况

本轮无老板级阻塞，无需打扰。
