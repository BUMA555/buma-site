01 · 搜索意图

工作流页的搜索意图是什么

不是所有访客都同一种意图。工作流页的访客通常在看完首页之后、决定是否联系之前——这个位置的页面，叫"转化前哨页"。

判断类意图（Informational） 想知道"这个 AI 团队怎么协作的""工作流是怎么跑的""我在这里充当什么角色"。他们不是来看功能列表，而是来找信任感。
比较类意图（Commercial Investigation） 正在比较 2-3 个 AI 团队或工具选项，想通过工作流页判断"谁的链路更透明、更适合一人公司"。
升级类意图（Transactional） 已经基本决定要找人协作，但需要先知道"合作后会先发生什么"。工作流页在这里的作用是消除最后的不确定感，而不是介绍功能。

02 · 标题 Meta 公式

工作流页的标题 Meta 怎么写

工作流页标题有两个常见错误：一是写成功能堆砌（"AI 工作流平台"），二是写成品牌独白（"我们是如何工作的"）。正确写法是：先写给谁看，再写解决什么问题，最后写能获得什么。

标题公式

主标题：给谁 + 什么协作场景 + 什么结果

示例：OpenClaw 一人公司工作流怎么搭｜先跑通内容、线索与执行闭环

副标题：不是先堆 Agent，而是先跑通最小链路

Meta Description 公式

第一句：解决什么问题（给谁）

第二句：怎么解决的（链路结构）

第三句：能获得什么（转化信号）

示例：给一人公司与小团队的 OpenClaw 工作流起步指南：先拆角色，再接消息、内容、FAQ 与客户承接，让 Agent 不是摆设，而是真能推进业务。

关键词布局 标题前 20 字内必须包含核心关键词（"OpenClaw 工作流""一人公司 AI 团队"）。Meta description 前 90 字内必须出现主关键词，同时保持可读性，不堆砌。
Slug 建议 用拼音或英文路径，保持语义清晰：`openclaw-one-person-workflow.html` 或 `ai-team-workflow-one-person-company.html`。
03 · 六模块结构

工作流页的标准六模块

参考 Linear / Stripe / Vercel 的 How It Works 页面，结合一人公司的实际情况，提炼出适合直接照抄的六模块结构。

模块一：入口判断（Who This Is For）

第一屏必须先说清楚"这页是给谁的"。不是给所有人看的，是给已经知道 AI Agent 概念、正在判断怎么落地的一人公司创始人。

写法要点：先描述 2-3 个典型卡点场景（"每次发文章要花 3 小时""想用 AI 但接不起来""接起来了但用不起来"），再写工作流解决的是哪个。

模块二：链路总览（The Flow at a Glance）

用 3-5 步把整个链路画出来，不讲细节，只讲顺序。让访客先有一个"我知道这怎么跑的"的全局感。

参考形式：① 消息入口接入 → ② 内容 Agent 执行 → ③ FAQ 承接 → ④ 客户分流 → ⑤ 转化闭环

模块三：角色分工（Role Breakdown）

说明在这个链路里，AI Agent 负责什么，人类（你）负责什么。这是消除"AI 会不会替代我"焦虑感的关键模块。

写法要点：不要只写"AI 负责内容生产"，要写"AI 负责第一稿生产，你负责判断质量方向和最终签发"。分工越具体，越能建立信任。

模块四：执行细节（How Each Step Works）

把链路总览里的每一环展开，讲清楚"实际发生时是什么样子"。每个步骤配一个场景描述。

示例（消息入口接入）："你在飞书 @ 机器人说'写一篇关于 SEO 的文章'，Agent 自动识别意图、调用内容工作流、生成大纲、交付初稿，全程你只需要判断方向是否正确。"

模块五：信任信号（Trust & Proof）

一人公司最容易缺的信任信号是"证明"，而不是"资质"。工作流页的信任信号要围绕"跑了多久""哪些场景验证过"来写，而不是堆证书和客户数量。

示例：这套工作流已在 OpenClaw 官方文档站验证，覆盖从消息接入到 SEO 发文完整链路。

模块六：单一 CTA（Single Next Action）

工作流页只允许一个主 CTA。工作流页访客的下一步通常是"我想先聊聊"，而不是"我要下载模板"。

CTA 写法建议：不是"开始使用 OpenClaw"，而是"先约 15 分钟，判断这链路适不适合你现在的情况"。后者更符合工作流页高意图但仍有疑虑的访客心态。

04 · Schema 双重标记

Article + BreadcrumbList 是工作流页的最小 schema 组合

工作流页不同于 FAQ 页（FAQPage）、解决方案页（Service）。工作流页本质是"我是怎么协作的"叙事，最适合的 schema 是 Article（文章/说明页），配合 BreadcrumbList 提供层级信号。

Article schema 要点 headline 必须与 <title> 一致；description 必须与 meta description 一致；datePublished 用首版发布时间；author 写具体负责人（不要只写品牌名）；keywords 列出 4-6 个核心词。
BreadcrumbList 要点 工作流页通常是三级路径：首页 → 文章中心（或解决方案） → 本页。三级 breadcrumb 对 Google 来说信号足够，不需要更多层级。
不需要加 FAQPage schema 工作流页不是 FAQ 结构，不要套 FAQPage。如果有高频问题，放在"信任信号"模块里用自然语言描述，而不是单独加 Q&A 结构化区块。
05 · 承接结构

工作流页和 FAQ、联系页的承接关系

工作流页是"看完就想聊"的页面，不是"看完还想继续翻"的页面。所以工作流页末尾的承接设计比内容本身更重要。

工作流页 → FAQ 页

如果访客看完工作流后还有操作层面的疑虑，要给他一个自助排查的出口。在 CTA 前加一行轻量引导："如果想先了解更多具体功能的接入方式，可以先看 OpenClaw FAQ。"

注意：FAQ 入口只在 CTA 前出现一次，不要在工作流页正文里反复出现，避免分流。

工作流页 → 联系页（主 CTA）

工作流页的 CTA 是所有页面里最接近"咨询转化"的一类。工作流页 CTA 不要写"联系我们"，要写"先约 15 分钟适配判断"，降低进入联系的门槛，同时筛选出真正有需求的访客。

参考话术："如果你看完这个链路觉得适合你现在的情况，可以先约 15 分钟，我们一起判断从这里开始最快能先解决什么问题。"

06 · 常见错误

工作流页的 5 个常见 SEO 错误

工作流页是最容易被写成"功能列表"或"品牌独白"的页面。以下 5 个错误是实际一人公司官网最常出现的问题。

错误一：标题写成产品功能列表 ❌ "OpenClaw AI Agent 工作流功能介绍" → 只有功能，没有场景和结果，Google 不知道这页解决什么问题。
错误二：Meta description 和首页重复 ❌ 直接复制首页的 description，导致 Google 认为这页是重复内容。
错误三：没有角色分工模块 ❌ 只讲"AI 做了什么"，不讲"人类负责什么"，访客会产生"AI 会不会替代我"的焦虑，反而增加咨询阻力。
错误四：信任信号堆证书而不是经验 ❌ 写"服务过 100+ 客户"，对一人公司反而显得不真实。改成"这套链路已跑通 90+ 篇 SEO 文章、覆盖 6 个城市主题"更有说服力。
错误五：CTA 超过 1 个 ❌ 同时出现"下载工作流模板""先预约咨询""了解更多功能"三个 CTA，等于没有 CTA。访客会因选择太多而什么都不点。
07 · 落地检查

工作流页 SEO 落地检查清单

文章写完或页面更新后，用这张清单逐项核对，不需要全懂，对着检查即可。

□ 标题前 20 字包含核心关键词 检查 <title>，确保"工作流"或"Workflow"出现在前 20 字。
□ Meta description 与首页 description 不重复 复制首页 description 到搜索框对比，完全相同则需要改写。
□ 有且只有 1 个主 CTA 搜索页面里所有 <a class="btn">，只能有 1 个主转化 CTA，其余都是次级导航。
□ Article schema headline = <title> 检查 JSON-LD 里的 headline 字段与 <title> 标签文字是否完全一致。
□ BreadcrumbList 三级路径完整 检查 BreadcrumbList itemListElement 有 3 项，且最后一项 name = 页面标题。
□ H1 与 title 一致或语义等价 <h1> 可以比 title 略长，但核心关键词要出现在 h1 里。
□ 内链指向 FAQ 页和联系页 正文里至少 2 处自然内链，分别指向 openclaw-faq.html 和 contact.html。
□ 移动端 CTA 按钮可点击且不重叠 用浏览器开发者工具切换到 mobile view，检查 CTA 按钮是否被导航栏遮挡。

08 · 风险提示

本方案的风险

工作流页 SEO 有两个主要风险，写方案时需要提前想清楚。

风险一：工作流页内容与官网文档高度重叠 如果 OpenClaw 官方文档已经很完整，工作流页如果只写"怎么配置"，Google 会判断这页是重复内容。 → 调整方向：一人公司官网工作流页侧重"我怎么跑通这条链路"，而不是"OpenClaw 功能怎么用"，用第一人称叙事代替功能说明。
风险二：工作流页 CTA 转化率低于预期 工作流页的访客通常是中等意图——他们想了解链路，但还没决定是否联系。如果 CTA 门槛设置过高，访客会关掉。 → 调整方向：主 CTA 用"先约 15 分钟"而非"立即咨询"，同时在 CTA 前增加一个次级选项："先看 FAQ" 或 "先看文章中心"，让不同意图层级的访客都有路可走。
