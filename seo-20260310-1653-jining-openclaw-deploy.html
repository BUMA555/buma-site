How To Start

济宁 OpenClaw 本地部署，建议先按这 4 步推进

先起底座，再定入口，再做岗位分工，最后再扩工作流。这更贴合 BUMA 一贯的主线：赋能、放大、补位、成事。

第一步：先把 Gateway 和控制界面跑起来

按公开文档把安装、onboard、Gateway 状态与 dashboard 跑通，先确认这套底座能稳定使用，再谈后续扩展。

第二步：只选一个消息入口

飞书、官网咨询、内部待办提醒都可以，但第一阶段只建议选一个入口，先保证收得到、看得清、分得动。

第三步：把 Agent 岗位边界写清楚

内容 Agent 负责初稿和整理，协同 Agent 负责提醒和流转，客服 Agent 负责高频答疑，人工负责承诺、判断和最终输出。

第四步：根据真实使用再加工作流

先跑稳一条链路，再扩更多渠道、知识库和更复杂的工作流。顺序对了，系统会越用越顺；顺序错了，只会越堆越乱。

What To Avoid

济宁做 OpenClaw 本地部署，最容易踩的 3 个坑

这些坑并不神秘，基本都来自部署顺序、岗位边界和协同入口没有提前理顺。

还没跑稳底座，就急着加很多渠道

Gateway、日志、控制界面和首个入口没稳定前，渠道越多越难排查，最后不是快，而是乱。

把部署理解成一次性装机

真正的部署不是装完就结束，而是要把消息入口、岗位边界和实际工作流一起梳理清楚。

过早承诺“全自动”

报价、对外承诺、合作决策和关键判断仍然需要人工把关。OpenClaw 更适合先补位、提速和稳定流程。
