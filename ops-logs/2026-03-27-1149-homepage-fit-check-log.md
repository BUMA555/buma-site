# OPS LOG: 一人公司主站 UI 持续优化（2026-03-27 11:49）

## 一、本轮目标

**本轮只做 1 个小任务**：给首页补一个“适合先聊 / 先去看文章”的资格筛选模块，减少无效咨询点击，让 CTA 更像商业网站里的过滤入口，而不是所有人都点同一个按钮。

## 二、快速学习了什么

**参考页面 1：Basecamp 首页**
- 看到的点：文案非常直白，不绕；会直接说“这是不是适合你”，减少理解成本。
- 可复用点：把“适合谁 / 不适合谁”前置，帮助访客先自我判断。

**参考页面 2：Linear 首页**
- 看到的点：价值表达常收敛成 3 个明确选择点，结构非常利于快速扫读。
- 可复用点：用 3 条资格描述做筛选，比长段解释更容易促发点击。

## 三、本轮落地改动

**改动文件**：
- `site-draft/index.html`
- `site-draft/styles.css`

**实际改了什么**：
1. 在首页 `What We Do` 与 `How It Works` 之间新增 `Quick Fit Check` 模块。
2. 拆成左右两张卡：
   - 左侧：`值得先聊`，承接到 `contact.html`
   - 右侧：`可以先看文章`，分流到 `seo.html`
3. 每张卡都用 3 条条件做快速判断，降低用户“我该不该点咨询”的犹豫。
4. 在 `styles.css` 中新增 `section-fit-check / fit-grid / fit-card / fit-list` 等样式，并补了移动端单列适配。

## 四、为什么这轮这样改

这是一个小但真实影响转化的动作：
- 首页原来有 CTA，但缺少“谁该点 / 谁先别点”的资格过滤。
- 商业站不是只会喊按钮，更要帮用户降低决策成本。
- 这类资格筛选能同时提升两件事：
  - 更适合的人更敢点咨询
  - 还没准备好的人被自然分流到文章中心继续养熟

## 五、验收证据

**已看参考**：
- `https://basecamp.com`
- `https://www.linear.app`

**本地可见证据**：
- `index.html` 中新增注释块：`<!-- FIT CHECK -->`
- 新增标题：`先判断值不值得聊，避免一上来就浪费时间`
- 新增两条分流 CTA：
  - `进入 15 分钟适配判断 →`
  - `先去文章中心补判断 →`
- `styles.css` 中新增样式段：`/* ── Fit Check ─────────────────────────────────────── */`
- 移动端补充：`@media (max-width: 900px)` 下新增 `.fit-grid { grid-template-columns: 1fr; }`

## 六、风险

**本方案风险**：当前“值得先聊 / 先别急着约”的文字仍是站方主观判断，不是真实用户语言；若后续访客反馈觉得“被筛掉”或语气偏硬，则把右侧文案改成更柔和的“更建议先看这 3 篇文章”。

## 七、下一步

下一轮优先考虑其中一个小任务：
1. 给首页 CTA 区补“提交后会收到什么”的结果承诺，进一步压低咨询顾虑。
2. 给文章中心补“按问题分流”的入口，而不是纯列表展示。
3. 给联系页补一个更像商务承接页的“准备材料清单”。
